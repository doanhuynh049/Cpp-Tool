The SmartSockets BusinessWorks Palette

The SmartSockets palette for Business Works is a collection of five (5) resources. Combined, these resource allow a user to create connections, establish message types, send and receive SmartSocket messages.

Installation

 

The SmartSockets BusinessWorks palette is contained within the ss-bwp.jar file. This file must be placed in the [../bw/5.1/lib/palette] directory – the absolute path to this location depends on where you installed the BusinessWorks distribution.

The SmartSockets palette depends on the SmartSockets Java API, contained within the ss.jar file. In order for the SmartSockets Java API to be made available to processes running within BusinessWorks, the ss.jar file must be copied to [../bw/5.1/hotfix/lib].

Note [6/11/2004]:

Currently, the SmartSockets palette for BusinessWorks also relies on a third file, ss-flavor.jar. This JAR contains an update to a BusinessWorks system file [TnsFlavor.property] normally located in the TIBCOxml.jar. The generally available [GA] distribution of BusinessWorks does not contain an updated version of this file. Until the GA BusinessWorks distribution is updated, ss-flavor.jar must also be copied to [../bw/5.1/hotfix/lib]. If TnsFlavor.property is not updated to support the SmartSockets palette for BusinessWorks, the palatte will not work - this condition is subject to change.

Usage

 

The minimum set of resources required for a successful SmartSockets deployment within BusinessWorks are a SmartSockets Connection, a SmartSockets Message Type, and one of the following: a SmartSockets Publisher, a SmartSockets Receiver or a Wait for SmartSockets Message.

Schemas

Each SmartSockets Message Type has a corresponding, automatically generated, schema. This schema is derived from the grammar specified in the Configuration tab of the SmartSockets Message Type resource at design-time. The schema represents a SmartSockets Message with a user-specified collection of field types. SmartSockets activity resources automatically incorporate the schema corresponding to the SmartSockets Message Type referenced in their Configuration tab. Because SmartSocket palette resources automatically incorporate a schema, the input-output editor are ignored.

N.B.: The automatically generated schemas are available anywhere within a project, whenever the choice to pick a schema is available.

SmartSockets Connection

 

A SmartSockets Connection is a shared resource required by all SmartSockets BusinessWorks process activities. Its suggested that users review the relevant sections of the SmartSockets Users Guide for an in-depth definition of terms. The SmartSockets Connection user-interface panel has two (2) tabs:

Configuration

The configuration tab has the following fields.

Field

Description

Name

The label for the SmartSockets Connection.

Description

Short description of the activity.

Unique Subject

The unique subject automatically created upon connection to the RTserver.

Server Names

The logical connection name.

Project

The SmartSockets project to which this connection belongs.

Advanced

The advanced tab initially consists of a table, with two (2) columns and zero (0) rows. Using the control buttons along the right side of the table, the user may add and remove rows to the table. Each row represents an option name, and value.

SmartSockets Message Type

 

A SmartSockets Message Type is a shared resource required by all SmartSockets BusinessWorks process activities. Its suggested that users review the relevant sections of the SmartSockets Users Guide for an in-depth discussion. The SmartSockets Message Type user-interface panel consists of a single tab:

Configuration

The configuration tab has the following fields:

Field

Description

Name

The Message Type name, and the label for the SmartSockets Message Type within BusinessWorks.

Description

A short description of the activity.

Type Name

A string which uniquely identifies the message type.

Identifier

A number (>0 for user-defined types) which uniquely identifies the message type.

Use Standard Message Type

When selected, the type-name will be interpreted as a standard message type, the identifier field will be switched to read-only. Once the form is completed the identifier and grammar fields will be filled in automatically.

Grammar

A string, representing the data fields present in the message. A schema is automatically generated from the grammar, and made available to the project. When a SmartSockets Message Type is referenced from a SmartSockets activity, the schema will be used as the input/output for the activity.

Schema Namespace

The namespace uniquely identifying the schema generated from the grammar. This field is informational only.

SmartSockets Publisher

 

The SmartSockets Publisher is an activity for sending SmartSockets messges. Its suggested that users review the relevant sections of the SmartSockets Users Guide for an in-depth definition of terms. The SmartSockets Publisher user-interface panel consists of a single tab:

Configuration

The configuration tab has the following fields:

Field

Description

Name

The label for the SmartSockets Publisher.

Description

A short description of the activity.

Destination

The destination property of the SmartSockets message identifies where the message is going.

SmartSockets Connection

A reference to a SmartSockets Connection resource.

SmartSockets Message Type

A reference to a SmartSockets Message Type resource. The SmartSockets Message Type resource exports a schema generated from a grammar; this schema is the input schema for the SmartSockets Publisher activity.
